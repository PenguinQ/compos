# Layouts ðŸªŸ
ComPOS utilize several components to do basic layouting, they are:

1. Page
2. Content
3. Header
4. Footer
5. Container

Some of this components are optional like Footer or Container, but some are needed to achieve the desired layout, in a nutshell the basic structure should look like this:

```jsx
<template>
  <Page>
    <Header>...</Header>
    <Content>
      <Container>...</Container>
    </Content>
    <Footer>...</Footer>
  </Page>
</template>
```

Just like I said previously, some of the components here are optional, they are:

1. Header
2. Footer
3. Container

An the others like;

1. Page
2. Content

Are needed to achieve the current layout, especially **Content**. **Page** are usually used once on the root of the view, something like:

```jsx
<template>
  <Page>
    <RouterView />
  </Page>
</template>
```

While **Content** must be used every time you create a view within a **Page**, something like:

```jsx
// Settings.vue
<template>
  <Header>...</Header>
  <Content>...</Content>
</template>

// ProductDetail.vue
<template>
  <Header>...</Header>
  <Content>...</Content>
</template>
```

So in short, the hierarchy must be like this:

```
- Page
  - Header
  - Content
    - Container
  - Footer
```
